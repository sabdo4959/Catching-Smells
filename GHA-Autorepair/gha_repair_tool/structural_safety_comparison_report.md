# GitHub Actions 수정 방법별 구조적 안전성 비교 보고서

## 📋 실험 개요

3가지 GitHub Actions 워크플로우 수정 방법의 **구조적 안전성**을 키 구조 검증을 통해 비교 분석했습니다.

### 🔧 비교 대상 방법
1. **Baseline**: 기존 ChatGPT 기반 단순 수정
2. **Two-Phase**: 2단계 복구 방식 
3. **GHA-Repair**: Guided Prompt 기반 수정 (우리의 제안 방법)

### 📊 검증 방법
- **키 구조 검증**: YAML 파일의 키 계층 구조만 비교 (값은 블랙박스 처리)
- **검증 대상**: 100개 파일 (각 방법별로 동일한 원본 파일 세트)
- **안전 기준**: 원본과 수정본의 키 구조가 동일하고, smell 수정용 키만 추가된 경우

- **절대적 안전 파일 수에서도 최고 성능** (6개 vs 3개 vs 2개)

## 💡 주요 인사이트

### 1. Guided Prompt의 효과성 입증
- **구조 보존 능력**: 다른 방법들이 실패한 파일에서도 원본 구조를 안전하게 유지
- **일관성**: 기본적으로 안전한 파일에서는 모든 방법과 동일한 성능 유지
- **독창성**: 3개의 고유 성공 파일을 통해 guided prompt의 차별화된 능력 증명

### 2. 실제 데이터의 도전과제
- **원본 파일 품질 문제**: 대부분의 파일이 YAML 구문 오류를 포함
- **파싱 오류 유형**: 
  - 중복 키 (duplicate key)
  - 매핑 값 오류 (mapping values not allowed)
  - 탭 문자 오류
  - 구문 오류 등

### 3. 방법별 특성
- **Baseline**: 가장 기본적인 수정, 구조 변경 위험 높음
- **Two-Phase**: 중간 수준의 안전성, baseline보다 50% 향상
- **GHA-Repair**: 최고 수준의 구조 보존, 가장 안전한 수정 방식
